## Предметная область
Современные маркетплейсы должны поддерживать множественные способы оплаты и интеграцию с различными банками-партнёрами. Для этого используют **платёжные системы**. Система должна поддерживать:
- Разные способы оплаты: кредитная карта, СБП, сервис бесконтактной оплаты.
- Разные платежные шлюзы: Сбер, Тинькофф, Альфа
## Проблема
Каждый способ оплаты может работать через разные платежные шлюзы, и наоборот. При реализации платёжной системы возникает необходимость создания отдельного класса для каждой комбинации «способ оплаты + банк»:

## Решение
Используя паттерн "Мост", мы разделим иерархию на две независимые части:
- Абстракция — способы оплаты
- Реализация — платежные шлюзы

## Инструкция по запуску
1. Первым дело необходимо установить MinGW-w64 на сайте: https://www.mingw-w64.org/downloads/
2. На сайте скачиваем архив winlibs-x86_64-posix-seh-gcc-15.2.0-mingw-w64ucrt-13.0.0-r6
3. Архив распаковываем и сохраняем на диске С
4. Путь C:\mingw64\bin добавляем в переменные среды PATH
5. Перезапускаем vs code, заходим в терминал и запускаем сборку build.bat
6. Запускаем приложение python payment_gui.py
