## Предметная область
Система вопрос-ответ по вселенной "Трансформеры: Прайм". Приложение предназначено для поиска характеристик персонажей (имя, фракция, занимаемая должность, альт-форма, оружие) 
на основе структурированных данных, хранящихся в реляционной базе данных.

## Проблема
При прямой реализации взаимодействия между пользовательским интерфейсом QAController и источником данных возникают следующие проблемы:
1. Громоздкий код: для работы с БД нужно писать sql-запросы, обрабатывать результаты и управлять соединениями. Это добавляет много лишнего кода.
2. Нарушение принципа разделения ответственности: код, который отвечает за отображение информации пользователю, будет зависеть от того, как устроена база данных (какие таблицы, какие колонки).
3. Риски при внесении изменений в систему: если потребуется поменять БД или подключить внешний API потребуется масштабный рефакторинг всех модулей, обращающихся к данным.
## Решение
Для решения данной проблемы применён архитектурный паттерн проектирования "Шлюз". В системе реализован класс CharacterGateway, который выступает единой точкой входа и полностью инкапсулирует доступ к источнику данных.
